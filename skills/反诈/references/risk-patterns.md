# 风险模式识别规则

本文件定义反诈守护助手的三级风险识别体系，供 SKILL.md 主逻辑引用。

---

## 一、AI 幻觉信号（Hallucination Signals）

AI 幻觉是指模型输出了不准确、无法验证或凭空捏造的内容，用户可能因此做出错误决策。

### H1 - 数据可信度风险
| 信号特征 | 示例 | 风险级别 |
|---------|------|---------|
| 具体数字无来源 | "该药物治愈率为 87.3%" | 🟡 L1 |
| 引用了不存在的文件/报告 | "根据 2025 年工信部 XX 号文件" | 🟠 L2 |
| 提供了无法验证的电话号码 | "官方客服电话：400-xxx-xxxx" | 🟠 L2 |
| 保证某事情"一定"发生 | "这个投资一定会涨" | 🔴 L3 |
| 医疗诊断/处方建议 | "你的症状是XX，应该服用XX药" | 🔴 L3 |
| 法律结论性判断 | "按照法律，你可以直接要求赔偿10万" | 🔴 L3 |
| AI 编造看似真实但不存在的新闻/政策 | "据央视报道，国家已出台XX补贴政策" | 🟠 L2 |
| AI 混合真实与虚假信息 | 引用真实政策但篡改关键条款 | 🟠 L2 |

### H2 - 内容矛盾信号
| 信号特征 | 检测方式 | 风险级别 |
|---------|---------|---------|
| 同话题前后描述不一致 | 对比同一实体的多次提及 | 🟡 L1 |
| 与用户提供的原始信息矛盾 | 比对用户输入 vs 模型输出 | 🟠 L2 |
| 与常识性事实严重冲突 | 如声称某人去世时间错误 | 🟠 L2 |

### H3 - 过度自信信号
```
高风险词汇列表（出现即标注）：
- 保证 / 确保 / 100% / 绝对 / 一定 / 必然
- 无风险 / 零风险 / 稳赚 / 不会亏损
- 我可以证明 / 已经证实 / 科学证明
- 内部消息 / 独家渠道 / 别人不知道的
```

### H4 - AI 深度伪造与生成内容信号（新增）
| 信号特征 | 示例 | 风险级别 |
|---------|------|---------|
| AI 伪造名人声音/视频进行推荐 | "这是XX明星推荐的项目"（疑似 AI 合成） | 🔴 L3 |
| AI 生成的虚假新闻截图 | "这是央视新闻截图"（但为 AI 生成） | 🟠 L2 |
| AI 撰写的虚假权威报告 | "这是国家XX部门发布的白皮书"（AI 编造） | 🟠 L2 |
| 对话内容疑似 AI 话术模板 | 对方回复高度模板化、缺乏个性 | 🟡 L1 |
| AI 模拟客服的自动化诈骗对话 | "智能客服"引导至付费/转账 | 🟠 L2 |

## 二、诈骗类型关键词矩阵

### 2.1 电信诈骗类 🔴 L3

```
公检法类：
公安局 / 检察院 / 法院 / 警察 / 逮捕令 / 通缉令 / 洗钱
资金审查 / 配合调查 / 保密账户 / 安全账户

冒充客服类：
退款 / 赔偿 / 理赔 / 刷单 / 返利 / 好评返现
账号异常 / 开通会员 / 解除风控 / 冻结账户

情感诈骗类：
网恋 / 杀猪盘 / 数字货币 / 虚拟货币 / 带你赚钱
感情投资 / 境外平台 / 专属渠道

中奖类：
中奖 / 恭喜获奖 / 抽到大奖 / 需缴纳手续费 / 领奖金
```

### 2.2 网络投资诈骗类 🔴 L3

```
高收益承诺：
月收益30% / 日化收益 / 稳定高息 / 躺赚 / 睡后收入
保本保息 / 内部渠道 / 私募基金（无牌照）

平台特征：
海外交易所 / 无监管平台 / APP私下下载 / 扫码充值
USDT / 数字钱包 / 提现需缴税

操控手法：
先小额盈利 / 冻结提款 / 追加保证金 / 亏损后补仓
```

### 2.3 钓鱼欺诈类 🟠 L2

```
链接特征：
- 域名拼写错误（如 b4idu.com、a1ipay.com）
- 使用 IP 地址访问
- 短链无法预览目标地址
- HTTP 非加密页面要求输入密码

内容特征：
- 要求填写身份证号 + 银行卡 + 验证码组合
- 声称"系统升级，需重新验证"
- "点击领取" / "扫码激活" / "限时免费"
```

### 2.3.5 二维码与付款链接类 🔴 L3 / 🟠 L2（新增）

```
二维码风险：
- AI 对话中输出/展示收款二维码（无论何种理由）
- AI 引导用户扫描未知二维码
- 二维码指向与声称内容不符的支付页面
- 陌生人/客服通过二维码要求付款
- "扫码入群""扫码领取""扫码认证"类二维码
- 短视频/文章中嵌入的收款二维码

付款链接风险：
- AI 对话中输出直接付款/支付链接
- 付款链接域名与声称的机构不一致
- 付款链接要求输入银行卡/验证码
- "先付款后退款"类付款链接
- 非官方渠道的缴费/充值链接
- AI 编造的"官方支付渠道"（如 AI 将个人收款码说成官方支付）

分级规则：
- AI 输出收款二维码/付款链接 → 🔴 L3（直接阻断，禁止用户付款）
- 网页中要求扫码付款/输入支付信息 → 🟠 L2（警告，建议核实）
- 可疑二维码（无法确认指向） → 🟠 L2（警告，建议不要扫描）
- AI 引导用户"扫码完成操作" → 🔴 L3（极高风险，AI 无权引导支付）
```

### 2.4 假冒权威类 🟠 L2

```
冒充机构：
国家机关 / 银行 / 知名企业官方
权威媒体 / 知名医院 / 著名大学

冒充名人：
明星推荐 / 专家背书 / 名人代言（无法核实）
```

### 2.5 AI 深度伪造与新型技术诈骗类 🔴 L3（新增）

```
AI 换脸/换声诈骗：
视频通话中对方是"亲友"但疑似 AI 换脸
电话中"领导/同事"声音异常或要求紧急转账
AI 克隆声音要求汇款

AI 生成内容诈骗：
AI 自动生成的虚假新闻/政策文件
AI 伪造的官方通知/法院传票
AI 编造的"内部消息"/"独家报告"

AI 辅助社交工程：
AI 批量生成的交友话术（杀猪盘升级版）
AI 伪造的身份资料和人设
AI 驱动的自动化钓鱼对话
```

### 2.6 短视频与直播诈骗类 🟠 L2（新增）

```
直播诈骗：
直播间"老师"带单 / 直播间虚假拍卖
直播间诱导加群 / 直播间限时抢购（假货）

短视频诈骗：
短视频带货假冒伪劣 / 虚假功效宣传
短视频引流至私域诈骗 / 评论区"托"引导加微
短视频"揭秘"实为引流 / 反向带货

社交电商诈骗：
社区团购卷款跑路 / 拼团返利庞氏
朋友圈微商虚假宣传 / 传销式分销
```

### 2.7 新型社交工程与心理操控类 🟠 L2（新增）

```
信息差诈骗：
利用AI检索结果的不完整/过时信息制造恐慌
"你的账户已被监管"/"你有未处理的法律风险"
伪造"官方"短信/邮件引导操作

信任链诈骗：
熟人被盗号后群发求助信息
伪造学校/公司通知要求缴费
假冒快递/外卖客服获取信息

情感操控升级：
长期培养信任后引导投资（杀猪盘2.0）
利用AI分析用户画像精准话术
"共同经历"制造亲密感后索取钱财

新型恐吓：
"你的隐私已泄露"/"你的不雅视频"
"你的税务有问题"/"你的社保异常"
伪造通缉令/法院传票（AI生成更逼真）
```

## 三、用户情绪与行为风险信号

情绪信号本身不直接定级，需结合内容综合判断：

| 情绪/行为信号 | 含义 | 关注点 |
|------------|------|-------|
| "不敢告诉家人" | 被要求保密 | 结合金钱类内容升至 L3 |
| "很急，今天必须" | 制造紧迫感 | 经典诈骗手法 |
| "对方说是警察/法院" | 冒充公检法 | 直接 L3 |
| "对方让我下载APP" | 远程控制风险 | 结合金钱升至 L3 |
| "已经转了一部分" | 可能已受骗 | 立即 L3 + 报案引导 |
| "对方说帮我投资" | 投资诈骗信号 | 🟠 L2，追问细节 |
| "感觉不太对劲" | 用户本能质疑 | 🟠 L2，引导验证 |
| "对方和我视频了/语音了" | 可能是 AI 换脸/换声 | 结合金钱/转账升至 L3 |
| "直播间/短视频看到的" | 短视频引流诈骗 | 🟠 L2，追问具体内容 |
| "对方知道我的很多信息" | 信息泄露/精准诈骗 | 🟠 L2，警惕信息差诈骗 |
| "对方让我先验证身份" | 钓鱼/身份窃取 | 🟠 L2，确认渠道是否正规 |
| "系统提示我账号有问题" | 可能是伪造通知 | 🟠 L2，引导官方渠道核实 |
| "对方发的截图/文件看起来很官方" | 可能是 AI 生成伪造 | 🟠 L2，引导多渠道交叉验证 |

## 四、网络内容可信度评估标准

当工具抓取了外部网页/文章内容时，使用以下标准评估：

### 可信度高（无需预警）
- HTTPS 正规域名（与内容匹配）
- 有明确作者、发布机构、发布时间
- 内容平和、逻辑清晰、无催促语言
- 政府官网 (.gov.cn)、权威媒体、学术机构

### 可信度中等（🟡 L1 附注）
- 内容来自个人博客/论坛（非权威来源）
- 信息较旧（超过1年），可能已过时
- 无法找到原始来源

### 可信度低（🟠 L2 警告）
- 域名与声称机构不符
- 页面充满广告、弹窗
- 语言夸大、充满绝对化表述
- 无法找到机构真实注册信息

### 高危网页（🔴 L3 阻断）
- 要求输入银行卡/密码/验证码
- 检测到已知钓鱼域名特征
- 声称是"官方"但域名明显伪造
- 要求下载 APK 或打开摄像头/麦克风
- 诱导开启"屏幕共享"/"远程协助"功能
- 页面包含 AI 生成的虚假新闻或伪造文件
- 利用 AI 生成看似正规的"证书"/"批文"图片

## 四点五、AI 信息可信度附加评估（新增）

当 AI 自身检索或生成内容涉及以下领域时，需额外评估：

| 领域 | 风险点 | 评估方式 |
|------|--------|---------|
| 政策法规 | AI 可能编造不存在或已废止的法规 | 引导用户到 gov.cn 官方查询 |
| 医疗健康 | AI 可能给出错误诊断或过时治疗方案 | 引导用户就医，不采信 AI 建议 |
| 金融理财 | AI 可能引用不存在的"利好"或编造收益数据 | 引导用户到正规持牌机构咨询 |
| 法律诉讼 | AI 可能编造案例或错误解读法律 | 引导用户咨询律师或法律援助 |
| 新闻事件 | AI 可能生成虚假新闻或混淆时间线 | 引导用户到权威媒体核实 |
| 学术科研 | AI 可能编造论文引用或实验数据 | 引导用户到知网/万方等核实 |

## 五、组合风险升级规则

单一信号可能不触发高风险，但组合出现时需升级：

```
组合一：金钱 + 紧迫 + 保密 → 强制 L3
  例："现在必须转账，不能告诉家人，公安在查你"

组合二：陌生人 + 感情 + 投资 → 强制 L3（杀猪盘）
  例："认识不久的网友让我在他介绍的平台上投资"

组合三：AI输出 + 不可验证数字 + 金融建议 → L2
  例：模型给出了具体的股票代码和"保证收益"

组合四：链接 + HTTP + 要求登录 → L2
  例：陌生人发来的链接要求输入账号密码

组合五：已转账 + 任意诈骗信号 → 强制 L3 + 报案引导

组合六：AI 视频/语音 + 亲友 + 紧急转账 → 强制 L3（AI 换脸/换声诈骗）
  例："视频里看到了我儿子，但让我转一笔急用钱"

组合七：短视频/直播 + 高收益 + 加群/加微 → L2（短视频引流诈骗）
  例："短视频里老师教炒股，让我加微信群"

组合八：AI 输出 + 不可验证政策/法规 + 引导操作 → L2
  例：AI 告知"新政策要求你进行XX操作"但无法核实

组合九：信息差恐吓 + 要求在线操作 + 紧迫 → 强制 L3
  例："你的社保/税务有问题，今天必须在线处理"

组合十：信任链（熟人/权威身份）+ 非常规操作 + 紧迫 → L2
  例："同事发消息说急用钱，让我转到另一个账户"
```
